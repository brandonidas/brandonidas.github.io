My First Google Code Challenge

I recently got invited to complete a Google Code challenge for an *ML position* at their Singapore office. It surprised me a little because I didn't expect to hear back at all. 45 minutes, two questions and on an online code editor. The true challenge was that I had never done a code challenge or practiced on leetCode. And I had 5 days to prepare.

Preparation (And learning Python)

Practicing on leetCode began, the easy questions were trivial but the 'medium' questions were more like 'impossible under a 45 minute time constraint'.

I started in Java but after looking at both Java and Python solutions - it became clear; brevity of Python made it ideal for a timed challenge. Perhaps also for interviewing.

The Challenge Itself

... was stressful. I spent around 8 minutes in a state of panic. NOTE: the challenge environment's test I/O supplied input LINE BY LINE rather than as a Solution Object like in leetCode. So you would have to know how to parse the input from command line through Python's sys.stdin. However, I was unprepared for this and my state of panic turned what would normally be a 4 minute speed read into an 8 minute scramble. 8 of 45 precious minutes.

The environment itself was quite poor. It disabled copy and paste but guranteed internal copy-paste function would be unhindered - this complete non-function. Pasting would usually return a random string of numbers eg "165149614651" or a copy from several copies ago. Here'a screenshot nonetheless;



I had two questions, both 30 and 20 points respectively, I would rate them both as somewhere in between leetCode's easy and medium. Though first was definitely harder than the second, I only managed to complete the first questions and got only 75% of the way through the second. (only to have the solution come to me as I took an afternoon stroll that day. Interesting how that happens.)

I'm not sure if I can post the questions or not, but I will say I found them on leetCode forums later so keep grinding!

After Action Review

As I said, the questions were already on leetCode, albeit as forum posts and not IDE problems, and more interestingly as previous interview questions from Amazon.

Constructive Action
